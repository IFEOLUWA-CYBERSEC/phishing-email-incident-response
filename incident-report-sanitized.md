# Incident Report — Phishing Email (Sanitized)

## Incident Overview
A phishing email was reported by a user and analyzed to determine malicious intent and potential impact. The message exhibited characteristics consistent with credential harvesting campaigns.

## Evidence Handling
- Original artifacts were handled in a controlled environment
- Archive passwords and raw evidence are stored privately
- Public documentation has been sanitized to remove sensitive data

## Key Findings
- External sending infrastructure identified
- Indicators matched known malicious patterns
- No evidence of successful exploitation confirmed at time of analysis

## Indicators of Compromise (Sanitized)
Refer to the `iocs/` directory for structured indicators.

## Assessment
- **Likelihood:** Medium–High
- **Potential Impact:** Credential compromise
- **Scope:** Limited to targeted recipients

## Recommendations
- Block identified infrastructure
- Reset credentials for affected users
- Improve phishing detection rules
- Conduct user awareness training

## Notes
This report is a sanitized version intended for public demonstration and portfolio use. Raw forensic artifacts are retained privately.
